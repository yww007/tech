Oracle 19c RAC 极速安装指南

2026年3月29日 · 技术文档

前言

Oracle Real Application Clusters (RAC) 是Oracle数据库的高可用性解决方案，本文将详细介绍从零开始安装Oracle 19c RAC的完整流程，包括环境准备、软件安装、配置优化等关键步骤。

一、环境准备

1.1 硬件要求

  • 服务器配置：至少2台服务器，每台服务器至少16GB内存，4核CPU
  • 存储要求：共享存储至少100GB，建议使用SAN或NAS
  • 网络要求：至少2个网络接口（公网和私网）

1.2 操作系统要求

  • 支持版本：Oracle Linux 7.x/8.x、RHEL 7.x/8.x
  • 内核版本：至少3.10.0-514.el7
  • 软件包：需要安装必要的开发工具和依赖包

1.3 网络配置

  • 公网IP：用于客户端连接
  • 私网IP：用于节点间通信
  • VIP：虚拟IP，用于故障切换
  • SCAN IP：单客户端访问名称IP

二、软件安装

2.1 下载Oracle软件

从Oracle官网下载Oracle 19c RAC软件包：

# 下载Oracle Database 19c
wget https://download.oracle.com/otn/linux/oracle19c/190000_Linux-x86-64.zip

# 解压软件包
unzip 190000_Linux-x86_64.zip -d /u01/app/oracle/product/19c/dbhome_1

2.2 安装Grid Infrastructure

Grid Infrastructure是RAC的基础设施，包括集群软件和ASM：

# 运行Grid Infrastructure安装程序
cd /u01/app/oracle/product/19c/grid
./runInstaller

2.3 安装Oracle Database软件

在所有节点上安装Oracle Database软件：

# 运行Oracle Database安装程序
cd /u01/app/oracle/product/19c/dbhome_1
./runInstaller

三、配置优化

3.1 内核参数优化

修改内核参数以优化Oracle性能：

# 编辑/etc/sysctl.conf
vi /etc/sysctl.conf

# 添加以下参数
kernel.shmmax = 68719476736
kernel.shmall = 4294967296
kernel.sem = 250 32000 100 128
fs.file-max = 6815744
net.ipv4.ip_local_port_range = 9000 65500
net.core.rmem_default = 262144
net.core.rmem_max = 4194304
net.core.wmem_default = 262144
net.core.wmem_max = 1048576

# 应用参数
sysctl -p

3.2 用户限制优化

修改用户限制以支持Oracle：

# 编辑/etc/security/limits.conf
vi /etc/security/limits.conf

# 添加以下限制
oracle soft nproc 2047
oracle hard nproc 16384
oracle soft nofile 1024
oracle hard nofile 65536
oracle soft stack 10240
oracle hard stack 32768

五、验证测试

5.1 集群状态检查

# 检查集群状态
crsctl check cluster

# 检查集群资源
crsctl status resource

5.2 数据库状态检查

# 检查数据库状态
srvctl status database -d orcl

# 检查实例状态
srvctl status instance -d orcl -n node1

5.3 连接测试

# 测试数据库连接
sqlplus / as sysdba

# 测试RAC连接
sqlplus system/password@orcl

六、常见问题

6.1 安装失败

如果安装失败，请检查：

  • 系统资源是否充足
  • 网络配置是否正确
  • 存储是否可访问
  • 依赖包是否完整

6.2 集群无法启动

如果集群无法启动，请检查：

  • 网络连接是否正常
  • 时间同步是否正确
  • 防火墙规则是否正确
  • 权限配置是否正确

6.3 性能问题

如果遇到性能问题，请检查：

  • 系统资源使用情况
  • 数据库参数配置
  • SQL语句优化
  • 存储I/O性能

七、总结

Oracle 19c RAC的安装需要仔细规划和准备，本文提供了从环境准备到生产部署的完整指南。通过遵循本文的步骤，您可以成功搭建一个高可用、高性能的Oracle RAC环境。
